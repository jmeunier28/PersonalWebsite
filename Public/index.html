Internship Experience

From the months of January 2015 to the end of June 2015 I worked on the Software Quality Assurance Team for Broadcom Limited.

I mainly worked in developing python scripts to automate manual testing on SoCs, which are used in ethernet modems. I also had to perform manual testing for cases which could not be automated. This gave me a lot of experience dealing with an embedded linux environment.

This was a challenging position because I had to develop different test cases for each feature based off of feedback from customers and our software dev team. These features could be very simple, i.e. USB, PCIe, Ethernet, etc. Nevertheless, they could become more complex when developing test scenarios for booting different types of root filesystems (i.e. NAND, NOR, Initramfs).

My team was responsible for automating these test cases in the most efficient way. Therefore, while I was at Broadcom I worked on a major automation framework change. This was a team effort to develop a framework that could allow for the development of cleaner and more scalable code. This was accomplished by utlizing python's object oriented features to add many abstraction levels to our code, accompanied by a unique configuration file that the QA engineer could edit as she/he tests different products.

Curriculum Based Video Application For Kids

The goal was to create a meangingful technological experience for children that could enhance their field trip or in class learning. In order to do that a team of 3 BU computer engineers as well as myself designed an Android Application that draws on machine learning and computer vision principles to identify animals in videos, and then creates an interactive game for kids that allows them to learn more about the animals they see.

The idea for this application was rooted in and encouraged by the participation in the Large Scale Visual Recognition Challenge 2015 by Image Net.

The app is structured in a game format, which includes an interactive video player with a multiple choice game mode. The client was designed using the Android Studio IDE, and was written in Java. The video detection capabilities (animal classification) was implemented using Python and OpenCV. Amazon Web Services supported our cloud delivery network and allowed for communication with our client.

For the Android App:

  • Was designed with kid friendly UI
  • Has parental control options to allow adults to limit their childs game play time
  • Implements interactive video player that fetches game data from the cloud

For the Animal Classifier:

  • Implemented in Python and utilizes OpenCV functions
  • Takes very large image dataset of different animals you wish to train the classifier on
  • Creates a mask for each image and uses this as an input to an RGB Histogram feature extractor
  • Data is split into training and testing data and then used to train a Random Forest Classifier
  • Once the classfier is trained it can take a video as input and track which animals are in each frame

Jello, A tool for Synthetic Biologists

Jello is a user friendly tool to help Synthetic Biologists at Boston University's Cidar Lab optimize and test their electronic design specifications with Cello.

Cello converts electronic design specifications of combinational logic to complete DNA sequences encoding transcriptional logic circuits that can be executed in bacterial cells. A database of transcriptional repressors, characterized in the Voigt lab, provide genetic NOT gates and NOR gates that can be composed into any logic function.

This Tool is truly unique in that gives the Biologists a way to automate and even optimize their designs, while providing the user with an easy to use interface.

Jello is written using Javascript and Nodejs, but it is able to utilize the python-shell node module to run python scripts inside of the node server. The python script is used to iterate through large text files and uses an algorithm to compute an optimal implementation to send to Cello for processing. Meaningful information is retrieved and extracted with the use of a python script as well and then it is displayed on the client side of our applicaton.

This software was challenging because I did not have any prior knowledge of synthetic biology principles. This project is a good example of my ability to take software design practices I have learned and to find good solutions to problems despite a lack of background in the particular topic.

Temperature Analysis Using Nodejs

An ongoing project part of EC544 coursework. This challenge was to program motes to sense and communicate ambient temperature to a central server that collects data from the motes and graphs it on a webpage. An arduino uno was used to sense data and an Xbee shield was used to send data to the server. On the server side, the Node.js platform was utilised to collect information from the serial port, average the incoming data from all nodes dynamically (as new nodes joined they would be detected and included in the average temperature calculation). Data is stored in a mongoDB database collection, but is also sent to the client side in real time via a web socket.

Once the data reaches the client the user can decide whether they want to view the current temperature or a historical plot of all average temperatures collected by the application and stored in the mongoDB database. In order to invoke this a JQuery button event is used to trigger the server to pull all information from the database. This data then gets plotted on the client side using CanvasJS to dynamically draw the graph.
